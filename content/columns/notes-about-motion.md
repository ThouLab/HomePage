# 動きのある静的サイト：Canvas と「落ち着き」の両立

DeepMind のサイトのように、背景に動きがあると「現代的」な印象が出ます。  
一方で、動きが強すぎると読みにくさや疲れに繋がることもあります。

## 実装の方針

- **背景の動きは情報ではなく雰囲気** に留める
- 文字の可読性を優先（コントラスト / 余白）
- **prefers-reduced-motion** を尊重する

## このサイトの実装

- Hero にのみ Canvas を配置
- 点と線のネットワークを低速で動かす
- モバイルでは密度を下げ、軽量化

## アクセシビリティ

動きを止めたい人向けに、OS設定の「視差効果を減らす」等に追従します。  
CSS と JS の両方で、`prefers-reduced-motion: reduce` をチェックしています。

[理念](philosophy/) にも設計方針をまとめています。
